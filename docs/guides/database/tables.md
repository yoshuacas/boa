# Creating Tables

Every table you create becomes a REST endpoint. No routes to configure, no controllers to write. Create a table, deploy, and it's queryable from your frontend.

```javascript
// This works as soon as the table exists
const { data } = await supabase.from('todos').select('*')
```

## DDL Is Auto-Committed

> **Warning:** Your database auto-commits every DDL statement immediately. There is no transaction rollback for `CREATE TABLE`, `ALTER TABLE`, or `DROP TABLE`. Always use `IF NOT EXISTS` / `IF EXISTS` to make statements safe to re-run. If a statement goes wrong, you must write a corrective migration to fix it.

## Primary Keys

Use TEXT with a UUID default:

```sql
id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text
```

Your database does not support `SERIAL` or `BIGSERIAL`. If you need integer IDs, use identity columns:

```sql
id BIGINT GENERATED BY DEFAULT AS IDENTITY (CACHE 1) PRIMARY KEY
```

## Creating a Table

```sql
CREATE TABLE IF NOT EXISTS users (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
  email TEXT UNIQUE NOT NULL,
  display_name TEXT,
  avatar_url TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## Relationships and Resource Embedding

Your database does not support foreign key constraints (`REFERENCES`). Instead, name your FK columns with the `_id` convention: `user_id`, `post_id`, `order_id`. pgrest-lambda uses this naming to auto-link related tables, enabling resource embedding queries from the frontend:

```javascript
// pgrest-lambda resolves user_id -> users, post_id -> posts automatically
const { data } = await supabase
  .from('comments')
  .select('*, posts(*), users(display_name)')
```

Document the relationship in a comment so it's clear to other developers:

```sql
CREATE TABLE IF NOT EXISTS todos (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
  user_id TEXT NOT NULL,  -- references users(id), enforced in app
  title TEXT NOT NULL,
  completed BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**If you skip the `_id` convention** (e.g., naming a column `author` instead of `author_id`), resource embedding won't work. You'll have to join manually in a Lambda function instead of using `select('*, users(*)')`.

## Indexes

Your database requires the `ASYNC` keyword for all index creation:

```sql
CREATE INDEX ASYNC IF NOT EXISTS idx_todos_user ON todos(user_id);
CREATE INDEX ASYNC IF NOT EXISTS idx_todos_created ON todos(created_at DESC);
```

Always index columns used in WHERE clauses, especially FK columns like `user_id`.

## Common Data Types

| Use | Type | Example |
|-----|------|---------|
| UUID primary key | `TEXT DEFAULT gen_random_uuid()::text` | `id` |
| Email | `TEXT UNIQUE NOT NULL` | `email` |
| Short text | `TEXT` | `title`, `name` |
| Long text | `TEXT` | `description`, `content` |
| Boolean | `BOOLEAN DEFAULT FALSE` | `completed`, `is_active` |
| Integer | `INTEGER` | `quantity`, `priority` |
| Decimal | `NUMERIC(10,2)` | `price`, `amount` |
| Timestamp | `TIMESTAMPTZ DEFAULT NOW()` | `created_at`, `updated_at` |
| JSON | `JSONB` | `metadata`, `settings` |

## Schema Patterns by App Type

### Todo / Productivity

```sql
CREATE TABLE IF NOT EXISTS users (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
  email TEXT UNIQUE NOT NULL,
  display_name TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS todos (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
  user_id TEXT NOT NULL,  -- references users(id), enables select('*, users(*)')
  title TEXT NOT NULL,
  completed BOOLEAN DEFAULT FALSE,
  priority INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX ASYNC IF NOT EXISTS idx_todos_user ON todos(user_id);
```

### Social / Feed

```sql
CREATE TABLE IF NOT EXISTS posts (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
  user_id TEXT NOT NULL,  -- references users(id), enables select('*, users(*)')
  content TEXT NOT NULL,
  image_key TEXT,
  like_count INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS comments (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
  user_id TEXT NOT NULL,  -- references users(id)
  post_id TEXT NOT NULL,  -- references posts(id), enables select('*, posts(*)')
  body TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS likes (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
  user_id TEXT NOT NULL,
  post_id TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE UNIQUE INDEX ASYNC IF NOT EXISTS idx_likes_unique ON likes(user_id, post_id);
CREATE INDEX ASYNC IF NOT EXISTS idx_posts_user ON posts(user_id);
CREATE INDEX ASYNC IF NOT EXISTS idx_posts_created ON posts(created_at DESC);
CREATE INDEX ASYNC IF NOT EXISTS idx_comments_post ON comments(post_id);
```

### E-Commerce

```sql
CREATE TABLE IF NOT EXISTS products (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
  name TEXT NOT NULL,
  description TEXT,
  price NUMERIC(10,2) NOT NULL,
  image_key TEXT,
  stock INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS orders (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
  user_id TEXT NOT NULL,  -- references users(id)
  status TEXT DEFAULT 'pending',
  total NUMERIC(10,2) NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS order_items (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
  order_id TEXT NOT NULL,  -- references orders(id), enables select('*, orders(*)')
  product_id TEXT NOT NULL,  -- references products(id), enables select('*, products(*)')
  quantity INTEGER NOT NULL DEFAULT 1,
  price NUMERIC(10,2) NOT NULL
);

CREATE INDEX ASYNC IF NOT EXISTS idx_orders_user ON orders(user_id);
CREATE INDEX ASYNC IF NOT EXISTS idx_order_items_order ON order_items(order_id);
```
