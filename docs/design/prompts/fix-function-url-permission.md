Fix Lambda Function URL 403 Forbidden error. Since October 2025, AWS requires a second resource-based policy (lambda:InvokeFunction with FunctionUrlAuthType NONE) for public Function URLs. The SAM template only gets the first permission (lambda:InvokeFunctionUrl) auto-generated. Need to: (1) Add AWS::Lambda::Permission resource to both SAM templates (plugin/templates/backend.yaml and cli/templates/backend.yaml), (2) Add a Function URL permission check to boa verify, (3) Add this to the PITFALLS.md doc. See GitHub issue #1.
