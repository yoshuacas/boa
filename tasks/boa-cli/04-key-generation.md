# Task 04: Key Generation Module

**Agent:** implementer
**Design:** docs/design/boa-cli.md

## Objective

Implement `cli/lib/keys.mjs` as a programmatic port of
`plugin/scripts/generate-keys.mjs`. Pure Node.js crypto,
zero dependencies.

## Target Tests

From `cli/__tests__/keys.test.mjs`:
- generateKeys(secret) returns { anonKey, serviceRoleKey }
- anonKey payload has role: "anon"
- anonKey payload has iss: "pgrest-lambda"
- anonKey has ~10-year expiry
- anonKey has iat field
- serviceRoleKey payload has role: "service_role"
- serviceRoleKey payload has iss: "pgrest-lambda"
- serviceRoleKey has ~10-year expiry
- HMAC-SHA256 signature verification passes for both keys
- Two calls produce structurally valid keys

## Implementation

Replace the stub in `cli/lib/keys.mjs` with the full
implementation per the design's Key Generation section.

Reference `plugin/scripts/generate-keys.mjs` for the
proven logic. The CLI version:

1. Exports `generateKeys(secret)` (not a CLI script).
2. Returns `{ anonKey, serviceRoleKey }` (not stdout JSON).
3. Uses `createHmac` from `node:crypto`.
4. Signs JWTs with HS256, base64url encoding.
5. Anon key: `{ role: 'anon', iss: 'pgrest-lambda',
   exp: now + 10years, iat: now }`.
6. Service role key: `{ role: 'service_role',
   iss: 'pgrest-lambda', exp: now + 10years, iat: now }`.

**Important:** The issuer must be `pgrest-lambda`, not `boa`.
This matches the existing `generate-keys.mjs` and is required
because pgrest-lambda verifies the issuer in its authorizer.

The helper functions `sign(payload, secret)` and `b64url(str)`
can be module-private.

## Acceptance Criteria

- All keys.test.mjs tests pass.
- Existing tests still pass.
- Keys generated by this module are interchangeable with
  keys from `plugin/scripts/generate-keys.mjs` (same
  structure, same issuer, same signing algorithm).
