Replace S3 bucket CORS `AllowedOrigins: '*'` with a configurable allowlist.

Context:
- cli/templates/backend.yaml:166-167 sets S3 CorsConfiguration AllowedOrigins to ['*'], allowing any origin to PUT/GET through presigned URLs.

Required changes:
1. Add an `AllowedOrigins` CloudFormation parameter (CommaDelimitedList, default empty string) to cli/templates/backend.yaml.
2. Use `Fn::If` so that when AllowedOrigins is empty, the entire CorsConfiguration block is omitted (same-origin uploads still work; only cross-origin is blocked). When populated, pass the list verbatim to AllowedOrigins.
3. Plumb the parameter through cli/commands/deploy.mjs and cli/lib/sam.mjs so users can set it via `.boa/config.json` (`allowedOrigins: ["https://app.example.com"]`).
4. Update .boa/config.json schema docs if they exist (docs/ARCHITECTURE.md or equivalent).
5. Post-deploy output in cli/commands/deploy.mjs: if AllowedOrigins is empty, print a note explaining how to configure it.

Acceptance:
- Default deploy (no allowedOrigins in config) produces a bucket with no CORS rules. Cross-origin fetch is blocked by the browser.
- Setting allowedOrigins: ["https://app.example.com"] and redeploying produces exactly that AllowedOrigins value on the bucket (verify with aws s3api get-bucket-cors).
