How It Works

BOA is a skill plugin for coding agents. It encodes patterns from hundreds of real AI-built backends into a single skill file that your agent reads and follows.

The Skill

A SKILL.md file your agent reads. Contains architecture patterns, deployment scripts, and rules that prevent common failures.

The Website

Documentation, pricing calculator, and install instructions for developers discovering BOA.

The Dashboard

Local HTML pages that connect to your AWS CLI to visualize and manage your deployed backend.

Full explanation: docs/how-it-works.md

The Stack

Every service in the BOA stack is serverless. No servers to patch, no capacity to plan.

Database
Aurora DSQL
Serverless PostgreSQL. SQL you already know. Scales to zero.
Auth
Amazon Cognito
Sign-up, sign-in, MFA. First 10,000 users free.
Compute
AWS Lambda
Node.js 20.x. 1M free requests/month.
API
API Gateway (REST)
Managed REST APIs with Cognito authorization.
Storage
Amazon S3
File uploads via presigned URLs. 5 GB free.
Hosting
AWS Amplify
Frontend hosting with CI/CD from Git.

Full details: docs/stack-overview.md

App Types

BOA includes architecture patterns for six common app categories.

Productivity

Todo lists, notes, project management. CRUD with user-scoped data and file attachments.

Social

Feeds, posts, likes, media sharing. Relational data with fan-out patterns and image uploads.

Real-time

Chat, collaboration, live updates. WebSocket connections with message persistence.

E-commerce

Catalogs, carts, orders. Transactional writes with inventory management.

SaaS

Multi-tenant apps, CRM, analytics dashboards. Tenant isolation with shared infrastructure.

IoT

Device tracking, telemetry, health monitoring. High-throughput writes with time-series queries.

FAQ

Common questions about BOA.

Is this free?

BOA itself is free and open source. The AWS services it deploys have their own free tiers, which cover most prototypes. See the pricing calculator for exact costs at your scale.

Why not Supabase?

BOA deploys serverless AWS infrastructure in your own account. It scales to zero (no $25/month minimum), you own the infrastructure, and it runs on services that handle millions of users without re-architecture.

Why Aurora DSQL and not DynamoDB?

SQL you already know, a relational model that fits most apps, and the PostgreSQL ecosystem of tools and libraries. DynamoDB requires learning a new data modeling paradigm that AI agents frequently get wrong.

Can I use this without a coding agent?

Yes. The SAM templates, Lambda handlers, and deploy scripts all work standalone. The skill file is just documentation that an agent can read — you can read it too.

Is this an official AWS service?

No. BOA is a community-driven open-source project from AWS. It is not an AWS service and has no SLA.
