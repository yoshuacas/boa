How we calculate these numbers

Per-user request model

Each app type has a characteristic requests-per-user-per-month (e.g., 1,000 for a todo app, 8,000 for a real-time chat app), split into reads and writes based on the app's access pattern.

Full-stack cost

We sum every AWS service in the path: API Gateway request, Lambda invocation + compute, DSQL database operations, Cognito MAU, and S3 storage. Nothing hidden.

Free tier applied

Each AWS service has its own free tier. We deduct all of them. Supabase's free plan limits (0.5 GB DB, 50K MAU, Nano compute) are applied the same way.

Conservative estimates

DSQL DPU-per-operation rates use the TPC-C benchmark (0.029 blended), which models complex real-world transactions. Simple key lookups cost less. Lambda assumes 256 MB / 100ms per invocation.

Supabase Pro auto-upgrade

Supabase free tier is limited to 0.5 GB DB and 50 ops/sec. When a workload exceeds those limits, the calculator upgrades to Pro ($25/mo base) with the cheapest valid compute tier.

What's not included

Data transfer, CloudWatch logs, and Amplify hosting costs are excluded (typically negligible at these scales). Supabase add-ons (PITR, phone MFA, custom domains) are also excluded.
