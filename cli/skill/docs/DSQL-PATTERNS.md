# DSQL Patterns

Aurora DSQL-specific patterns for the BOA stack.

---

## DSQL Limitations

These are the hard constraints of Aurora DSQL. Every migration and schema the agent writes MUST follow these rules.

### DDL Constraints

| Feature | DSQL Support | Workaround |
|---------|-------------|------------|
| `SERIAL` / `BIGSERIAL` | **Not supported** | Use `TEXT DEFAULT gen_random_uuid()::text` for UUIDs, or `BIGINT GENERATED BY DEFAULT AS IDENTITY (CACHE 1)` for auto-increment |
| `REFERENCES` (foreign keys) | **Not supported** | Enforce referential integrity in application logic; document relationships in comments |
| `CREATE INDEX` | **Must use `ASYNC`** | Always `CREATE INDEX ASYNC IF NOT EXISTS` |
| Identity columns | **Must be `BIGINT`** | Cannot use `INTEGER` identity; `CACHE` clause is required |
| Stored procedures / functions | **Not supported** | Implement in Lambda handlers |
| Triggers | **Not supported** | Implement in Lambda handlers |
| Row-Level Security (RLS) | **Not supported** | Use Cedar authorization policies (see POLICIES.md) |
| `SET ROLE` | **Not supported** | Cedar policies handle row-level filtering |
| Advisory locks | **Not supported** | Use application-level locking |
| Sequences (standalone) | **Not supported** | Only identity columns provide auto-increment |
| Multi-statement queries | **Not supported** | Execute one SQL statement at a time |
| `TRUNCATE ... RESTART IDENTITY` | **Not supported** | Use `DELETE FROM` to clear tables |
| Transactional DDL | **Not supported** | DDL is auto-committed; use `IF NOT EXISTS` for safety |

### Supported Features

- Unique constraints
- CHECK constraints
- NOT NULL constraints
- `gen_random_uuid()` function
- `NOW()` / `CURRENT_TIMESTAMP`
- Standard DML (SELECT, INSERT, UPDATE, DELETE) in transactions
- GIN indexes (for full-text search)

### IAM Authentication

- Auth tokens expire in **15 minutes**
- Refresh tokens every **10 minutes** in long-running processes
- Single database per cluster: always `postgres`

## Connecting to DSQL from Lambda

DSQL uses IAM authentication. Never hardcode credentials.

```javascript
import { DsqlSigner } from '@aws-sdk/dsql-signer';
import pg from 'pg';

const { Pool } = pg;

const endpoint = process.env.DSQL_ENDPOINT;
const region = process.env.REGION_NAME;

let pool;

async function getPool() {
  if (pool) return pool;

  const signer = new DsqlSigner({ hostname: endpoint, region });
  const token = await signer.getDbConnectAdminAuthToken();

  pool = new Pool({
    host: endpoint,
    port: 5432,
    user: 'admin',
    password: token,
    database: 'postgres',
    ssl: { rejectUnauthorized: true },
    max: 5,
    idleTimeoutMillis: 60000,
    connectionTimeoutMillis: 5000,
  });

  // Refresh token before expiration (tokens valid 15 min)
  setInterval(async () => {
    const newToken = await signer.getDbConnectAdminAuthToken();
    pool.options.password = newToken;
  }, 10 * 60 * 1000);

  return pool;
}
```

## Schema Patterns for DSQL

### Primary keys — always use TEXT with UUID

```sql
id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text
```

Never use `SERIAL`, `BIGSERIAL`, or `INTEGER` primary keys.

### Relationships — use comments instead of REFERENCES

```sql
-- 001_create_users.sql
CREATE TABLE IF NOT EXISTS users (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
  email TEXT UNIQUE NOT NULL,
  display_name TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 002_create_todos.sql
CREATE TABLE IF NOT EXISTS todos (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
  user_id TEXT NOT NULL,  -- references users(id), enforced in app
  title TEXT NOT NULL,
  completed BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Indexes — always ASYNC

```sql
CREATE INDEX ASYNC IF NOT EXISTS idx_todos_user ON todos(user_id);
CREATE INDEX ASYNC IF NOT EXISTS idx_todos_created ON todos(created_at DESC);
```

### Auto-increment integer IDs (when needed)

```sql
CREATE TABLE IF NOT EXISTS events (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY (CACHE 1) PRIMARY KEY,
  event_type TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## Common Query Patterns

### Pagination (cursor-based)

```sql
SELECT * FROM items
WHERE user_id = $1
  AND created_at < $2
ORDER BY created_at DESC
LIMIT 20;
```

### Full-text search

```sql
-- Add a tsvector column
ALTER TABLE posts ADD COLUMN IF NOT EXISTS search_vector tsvector;
CREATE INDEX ASYNC IF NOT EXISTS idx_posts_search ON posts USING gin(search_vector);

-- Search
SELECT * FROM posts
WHERE search_vector @@ plainto_tsquery('english', $1)
ORDER BY ts_rank(search_vector, plainto_tsquery('english', $1)) DESC
LIMIT 20;
```

### Aggregation with materialized counts

```sql
-- When adding a like
BEGIN;
  INSERT INTO likes (user_id, post_id) VALUES ($1, $2);
  UPDATE posts SET like_count = like_count + 1 WHERE id = $2;
COMMIT;
```
